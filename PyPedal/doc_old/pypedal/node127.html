fast_reorder(myped, filetag='_new_reordered_', io='no', debug=0) list
func]pyp_utils!fast_reorder() fast_reorder() renumbers a pedigree such that parents precede their offspring in the pedigree. In order to minimize overhead as much as is reasonably possible, a list of animal IDs that have already been seen is kept. Whenever a parent that is not in the seen list is encountered, the offspring of that parent is moved to the end of the pedigree. This should ensure that the pedigree is properly sorted such that all parents precede their offspring. myped is reordered in place. fast_reorder() uses dictionaries to renumber the pedigree based on paddedIDs.
myped
A PyPedal pedigree object.
filetag
A descriptor prepended to output file names.
io
Indicates whether or not to write the reordered pedigree to a file (yes|no).
debug
Flag to indicate whether or not debugging messages are written to STDOUT.
Returns:
A reordered PyPedal pedigree.

filetag

renumber(myped, filetag='_renumbered_', io='no', outformat='0', debug=0, missingparent=0) list
func]pyp_utils!renumber() renumber() takes a pedigree as input and renumbers it such that the oldest animal in the pedigree has an ID of '1' and the n-th animal has an ID of 'n'. If the pedigree is not ordered from oldest to youngest such that all offspring precede their offspring, the pedigree will be reordered. The renumbered pedigree is written to disc in 'asd' format and a map file that associates sequential IDs with original IDs is also written.
myped
A PyPedal pedigree object.
filetag
A descriptor prepended to output file names.
io
Indicates whether or not to write the renumbered pedigree to a file (yes|no).
missingparent
Indicates the sire/dam ID used to identify unknown parents.
outformat
Flag to indicate whether or not to write an asd pedigree (0) or a full pedigree (1).
returnmap
Indicates whether or not an ID map, as well as the renumbered pedigree, should be returned. If nonzero, suppresses writing of ID maps to disc.
debug
Flag to indicate whether or not progress messages are written to stdout.
Returns:
A reordered PyPedal pedigree.

reorder(myped, filetag='_reordered_', io='no', missingparent=0) list
func]pyp_utils!reorder() reorder() renumbers a pedigree such that parents precede their offspring in the pedigree. In order to minimize overhead as much as is reasonably possible, a list of animal IDs that have already been seen is kept. Whenever a parent that is not in the seen list is encountered, the offspring of that parent is moved to the end of the pedigree. This should ensure that the pedigree is properly sorted such that all parents precede their offspring. myped is reordered in place. reorder() is VERY slow, but I am pretty sure that it works correctly.

filetag
A descriptor prepended to output file names.
io

missingparent
